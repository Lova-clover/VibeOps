Executive Summary

VibeOps는 바이브 코딩에 5단계 품질 파이프라인을 도입합니다.

1. 문제 정의

핵심 문제: 바이브 코딩에 품질 관리 프로세스가 없다
전통적 개발은 요구사항 → 설계 → 개발 → 테스트 → 리뷰 과정을 거칩니다.
바이브 코딩은 "만들어줘" → 코드 → 커밋으로 끝나는 경우가 많습니다.

2. 해결책: VibeOps 5단계 파이프라인

핵심: 코드 생성 "전"에 스펙 확정, 생성 "후"에 자동 검증, 전 과정 자동 문서화

Stage 1: Spec Gate (스펙 게이트)

모호한 자연어 요청을 구조화된 스펙으로 변환합니다. 개발자는 코드가 아닌 스펙을 확인하여 의도 불일치를 조기 발견합니다.

# "로그인 만들어줘" → 자동 생성 스펙 spec: feature: 사용자 로그인 requirements: [이메일/비밀번호 인증, JWT 토큰, bcrypt 해싱] constraints: [C001: OAuth 금지]

Stage 2: Verify Gate (검증 게이트)

새 요청이 기존 정책과 충돌하는지 코드 생성 전에 감지합니다.

⚠️ 충돌 감지: "소셜 로그인 추가" → [C001] OAuth 금지 정책과 충돌
대안: 자체 인증으로 구현 또는 정책 예외 추가 후 진행

Stage 3: Generate (코드 생성)

확정된 스펙 기반으로 AI가 코드 생성. 기존 결정, 기술 스택, 코딩 컨벤션이 자동 주입됩니다.

Stage 4: Validate Gate (품질 게이트)

생성된 코드의 정책 준수, 보안 취약점, 코드 일관성을 자동 검증합니다.

🛡️ 정책 검사

금지 키워드/패턴 포함 여부

🔒 보안 검사

하드코딩 비밀, SQL 인젝션 등

🎨 일관성 검사

기존 코드 스타일/패턴 일치

📋 스펙 대조

생성 코드가 스펙과 일치하는지

Stage 5: Auto Document (자동 문서화)

스펙, 결정, 검증 결과가 .vibe 디렉토리에 자동 저장됩니다.

# 코드 맥락 조회 $ vibe why src/auth/login.py → 스펙: SPEC-001 | 결정: D001 JWT 사용 | 제약: C001 OAuth 금지
