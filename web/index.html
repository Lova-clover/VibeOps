🧠 VibeMemory

AI 컨텍스트 영속화 시스템 - 바이브 코딩의 컨텍스트 증발 문제를 해결합니다

프로젝트 컨텍스트 (.vibe 디렉토리)

context.yaml

project: name: FinanceAPI description: B2B 금융 서비스 백엔드 tech_stack: language: Python 3.11 framework: FastAPI database: PostgreSQL 15

constraints.yaml

constraints: - id: C001 rule: OAuth 사용 금지 severity: critical keywords: - oauth - google.oauth2 - id: C002 rule: 외부 CDN 사용 금지 keywords: - cdn.jsdelivr

decisions.yaml

decisions: - id: D001 title: 인증 방식 선택 decision: 자체 JWT 인증 reason: 보안 정책 - id: D002 title: 비밀번호 해싱 decision: bcrypt 사용

🛡️ Constraint Guard - 금지 규칙 위반 감지

코드를 입력하면 정책 위반 여부를 실시간으로 검사합니다.

🤖 Context Injection - AI 프롬프트 자동 생성

VibeMemory가 생성하는 AI 시스템 프롬프트입니다. 이 프롬프트를 AI 세션에 주입하면 컨텍스트가 유지됩니다.

# 프로젝트 컨텍스트 (VibeMemory) ## 프로젝트: FinanceAPI B2B 금융 서비스 백엔드 API ## 기술 스택 - language: Python 3.11 - framework: FastAPI - database: PostgreSQL 15 ## ⛔ 금지사항 (절대 위반 금지) - [C001] OAuth/소셜 로그인 사용 금지 이유: 보안 정책상 외부 IdP 연동 불가 - [C002] 외부 CDN 사용 금지 이유: 폐쇄망 배포 예정 ## 📋 기술 결정 사항 - [D001] 인증 방식: 자체 JWT 인증 - [D002] 비밀번호 해싱: bcrypt 사용
💉

Context Injection

AI 세션 시작 시 프로젝트 맥락을 자동으로 주입합니다. 매번 설명할 필요가 없습니다.

🛡️

Constraint Guard

금지 규칙 위반을 실시간으로 감지합니다. OAuth 금지인데 OAuth 코드 생성 시 즉시 경고합니다.

📝

Decision Capture

기술/아키텍처 결정을 자동으로 기록합니다. 왜 이 방식을 선택했는지 추적 가능합니다.

🔍

Intent Tracking

생성된 코드와 원래 의도를 연결합니다. 3개월 후에도 맥락을 복원할 수 있습니다.
