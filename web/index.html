😤 바이브 코딩의 진짜 문제: 컨텍스트 증발

AI는 대화 내용을 기억하지 못합니다. 이것이 바이브 코딩에서 발생하는 구조적 문제입니다.

09:00 - 프로젝트 시작

"OAuth는 절대 쓰지 마. 보안 정책이야"

09:30 - 로그인 구현

"JWT 기반 자체 인증으로 구현했습니다" ✓

10:30 - 50번의 대화 후...

"소셜 로그인도 추가해줘"

10:31 - AI의 응답 🚨

"Google OAuth를 추가하겠습니다!"
→ 초기 금지사항 완전히 잊음

🔥 왜 이런 일이 발생하는가?

  • AI는 128K~200K 토큰의 단기 기억만 보유
  • 대화가 길어지면 초기 내용이 선입선출로 삭제
  • 새 세션 시작 시 이전 맥락 100% 소멸
  • SonarQube, ESLint로 해결 불가 - 바이브 코딩만의 문제

💉 Context Injection - 핵심 기능

AI 세션 시작 시 프로젝트 맥락을 자동으로 주입합니다.
더 이상 매번 "이 프로젝트는 FastAPI 쓰고, OAuth는 금지야"라고 설명할 필요 없습니다.

→
🔄 VibeMemory 파싱
→
📋 시스템 프롬프트 생성

🤖 AI에 자동 주입

📁 입력: .vibe 디렉토리

# .vibe/context.yaml project: name: FinanceAPI description: B2B 금융 서비스 백엔드 tech_stack: language: Python 3.11 framework: FastAPI database: PostgreSQL 15

🤖 출력: AI 시스템 프롬프트

# 프로젝트 컨텍스트 (VibeMemory) ## 프로젝트: FinanceAPI B2B 금융 서비스 백엔드 API ## 기술 스택 - language: Python 3.11 - framework: FastAPI - database: PostgreSQL 15 ## ⛔ 절대 금지사항 - [C001] OAuth/소셜 로그인 사용 금지 - [C002] 외부 CDN 사용 금지 ## 📋 기술 결정 사항 - [D001] 인증: 자체 JWT - [D002] 해싱: bcrypt 사용

💡 이게 왜 중요한가?

❌ VibeMemory 없이

매 세션마다 5분씩 설명:

"이 프로젝트는 FastAPI로 만들고 있고, PostgreSQL 쓰고..."

→ 하루 2시간 낭비

✅ VibeMemory 사용

AI가 이미 알고 있음:

"네, FinanceAPI 프로젝트네요. 어떤 기능을 구현할까요?"

→ 0초, 바로 시작

🛡️ Constraint Guard - 정책 위반 실시간 감지

금지 규칙을 정의하면, AI가 생성하는 코드에서 위반 사항을 실시간으로 감지합니다.

규칙 정의

constraints: - id: C001 rule: OAuth 사용 금지 keywords: - oauth - google.oauth2 - id: C002 rule: 외부 CDN 사용 금지 keywords: - cdn.jsdelivr

코드 검사 테스트

📝 Decision Capture - 기술 결정 자동 기록

AI와의 대화에서 기술/아키텍처 결정을 자동으로 추출하여 기록합니다.

💬 AI와의 대화

👤 개발자

비밀번호 해싱은 어떤 알고리즘이 좋을까?

🤖 AI

bcrypt를 추천합니다. 솔트가 자동 생성되고, 업계 표준입니다.

👤 개발자

좋아, bcrypt로 하자. argon2는 왜 안 써?

🤖 AI

argon2가 더 최신이지만, 일부 환경에서 설치 이슈가 있습니다.

📋 자동 추출 결과

# .vibe/decisions.yaml (자동 생성) decisions: - id: D003 title: 비밀번호 해싱 알고리즘 decision: bcrypt 사용 alternatives_rejected: - name: argon2 reason: 설치 이슈

✅ 3개월 후에도 "왜 bcrypt?"에 답 가능

🔍 Intent Tracking - 의도 추적

vibe why 명령으로 "왜 이렇게 만들었는지" 즉시 조회합니다.

🤔 신입 개발자의 질문

👨‍💻 신입 개발자입사 1주차

"이 login.py 왜 이렇게 복잡해요?"

❌ 없이

vibe why login.py
